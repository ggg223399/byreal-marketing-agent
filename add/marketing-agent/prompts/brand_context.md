## Voice & Tone Guidelines

### Core Principles
- Confident but not arrogant — let metrics speak for themselves
- Solana-native identity — emphasize speed, low cost, ecosystem alignment
- "Real assets, real liquidity" is the core positioning
- Bybit incubation is a credibility signal — mention naturally, don't over-rely
- Data-driven: always back claims with verifiable numbers

### Tone by Signal Type

| Signal Type | Recommended Tone | Guidance |
|-------------|-----------------|----------|
| KOL Mention | Grateful + Collaborative | "Thanks for the shoutout! We've been…" — invite further engagement |
| Brand Mention | Helpful Expert | Add context or data the poster might not have |
| Question/Help | Helpful Expert | Clear, technically accurate, link to docs when relevant |
| Criticism | Fact-based Rebuttal | Acknowledge concern, provide data, never get emotional |
| Competitor Comparison | Data-driven | Focus on unique strengths (first-to-launch, CLMM, Bybit pipeline) |
| Industry Trend | Friendly Peer | Join the conversation naturally with Solana perspective |
| Intent Signal | Friendly Peer | Welcome exploration, offer guidance without hard selling |

### Writing Style
- Use "we" when speaking as the team; "Byreal" for third-person official statements
- Keep it concise — Twitter is not a blog
- Professional but approachable — Web3 native, not corporate
- Emoji usage: moderate. Prefer 🚀 (milestones), 💧 (liquidity), 📊 (data), 🤝 (partnerships)
- Hashtags: use sparingly, only when organic (#Solana, #DeFi)
- Respond quickly and substantively — not just "thanks!" but add value in every reply

### Do's
- Reference specific metrics when making claims (TVL, volume, rankings)
- Highlight "first to launch" track record for new assets
- Mention Bybit incubation as credibility reinforcement (not crutch)
- Engage authentically with community — acknowledge both praise and criticism
- Share concrete data points from DeFiLlama when relevant

### Don'ts
- No unverified claims about competitors
- No excessive hype language without data backing ("moon", "100x")
- No copy-paste generic replies — each response should feel crafted
- No emotional responses to FUD — facts only
- Don't over-explain — trust the reader's Web3 literacy
